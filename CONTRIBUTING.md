# Contributing to AI-Playbook

Thank you for contributing to the AI-Playbook.

## Getting Started

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with clear messages
5. Push and open a Pull Request

## Contribution Guidelines

### Content Standards

- **Be specific**: Provide concrete examples, not abstract descriptions
- **Cite sources**: Link to references and official documentation
- **Include trade-offs**: Document pros and cons when relevant
- **Update related content**: Cross-reference related entries

### File Organization

- Use lowercase with hyphens for file names
- Include README.md in each folder
- Follow existing templates
- Keep files focused on a single topic

### Documentation Style

- Use active voice
- Front-load important information
- Include code examples where applicable
- Add tables for comparisons when helpful

### Commit Guidelines

- Use conventional commits: `type(scope): description`
- Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`
- Keep commits focused and reviewable
- Reference issues where applicable

## Pull Request Process

1. Update relevant documentation
2. Ensure all checks pass
3. Request review from maintainers
4. Address feedback constructively

## Content Categories

### Adding Knowledge
Place new entries in `knowledge/` following the established format.

### Adding Prompts
Add prompt templates to `prompts/` organized by capability.

### Adding Agents
Define agents in `agents/` with complete configuration.

### Architecture Decisions
Create ADRs in `adr/` using the template for significant changes.

## Questions?

Open an issue for discussion before starting large contributions.
